# Реализация тестового задания на базе шаблон для выполнения тестового задания

## Описание
В Google Cloud Console создайте проект,
включите Google Sheets API, 
создайте учетные данные 
(OAuth 2.0 или сервисный аккаунт) и скачайте файл,
файл нужно переименовать и разместить по пути
```bash
/.credentials/google-credentials.json
```

дальше нужно создать .env по примеру examole.env
В нем нужно заполнить только 2 поля 
```bash
WB_API_TOKEN : WB API токен
```
```bash
GOOGLE_SHEET_IDS=sheet1,sheet2
```
список таблиц в который записывать тарифы. 

ID таблицы присутствует в URL: **1h7LN3UQRyd0xEISrnhkRcaDfW7qH2XJUmy1_qWPSClU**
`https://docs.google.com/spreadsheets/d/1h7LN3UQRyd0xEISrnhkRcaDfW7qH2XJUmy1_qWPSClU/edit?
`

После этого можно запустить проект:
```bash
docker compose up --build
```

## Краткий сценарий
 - при запуске контейнера сразу выполнится импорт тарифов и экспорт в таблицы, а дальше по крон расписанию.
 - импорт выполняется с retry, 5 попыток получить данные => успех или лог текст ошибки
 - запись в базу выполняется чанками по 500 строк (если будет больше 500), нак как список values строится в ram, импорт из тысяч строк может привести к падению
 - запись в google-sheet`s, главная сложность: таблиц N, тарифов N, таблицы и тарифы тоже тянем из базы и обрабатываем чанками (limit,offset) до 500 строк.
 - - с помощью генераторов обходим таблицы и на каждую итерацию пишем в каждую таблицу тарицы чанками, это позволяет сохранить консистентность каждой таблицы, и недопустить вывода всех таблиц наполовину заполнеными если скипт упал.
